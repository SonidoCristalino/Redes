    Capas: 5 para internet. 
        Aplicación, transporte, red, enlace de datos, física. Se divide para
        poder alcanzar el problema completo de internet y asignarle ciertos
        servicios a cada uno de ellos.  Cada uno ofrece determinados servicios
        dependiendo la capa. 

    Se puede equiparar con el servicio de línea area. Para poder abordar el
    problema se puede hacer más fácil dividiéndolo en capas. 
    Cuando se compra la el pasaje por ejemplo se determina en determinada capa,
    las maletas para otra capa, la pista de despegue y navegación lo administra
    otra capa, etc. 
    Cada capa es transparente para todas las demas, cada capa no se entera de
    otras circunstancias propias de la otra; se trata de que si se modifica algo
    de una capa no modificar todas las demas. 
    Las capaas es un modelo para entender la red física, es un marco teórico
    para que los individuos puedan hacer el sistema ma versátil posible. 
    Se requiere una estructura para poder establecer la relación entre
    capas,para ello se requiere la modularización. Cada capa se comunica con la
    misma capa indpendientemente de las demás. 

    En interne, conexiones modernas, el modelo es el mismo, de cómo trabajan las
    comunicaciónes, es un modelo TCP/IP. Este modelo ayuda tanto para los
    usuarios, estudiantes, empresarios, etc. 
    Toda la problemática se divide en 5 capas ¿por qué esas cinco? Si se tienen
    100 capas, se dividen mcuhos problemas chicos no lleva a ninguna solución. 
    Tener pocas capas quita el principio de dividir el problema, 5 son las que
    terminan balanceando el problema entre dividir y entender. 

    Capa Aplicación : donde residen las aplicacione sy protocolos. Ejemplo: el
    Skype de un host que se comunica con otro Skype de otro host. Skype no
    piensa en las otras capas, sólo en comunicarse con el otro usuario.  Los
    protocolos más famosos FTP para transferrir archivos, el protocolo
    especifica cada vez que se envía un archivo; SMTP de correo, cuando se
    tansmite mediante ese protocolo no se transmite de forma arbitraria, sino
    como se especifica el protocolo, cuando se quiere enviar un mail se tansmite
    mediante ese protocolo no se transmite de forma arbitrariao como se
    especifica el protocolo. Es por ello que se puede enviar mail entre
    distintos servidores como Yahoo y Gmail. Protocolo HTTP es el que va a
    marcar el protocolo de cómose dben presentar las páginas webs, qué pasa si
    me trato de conectar, etc. Muchas veces se utilizan dos protocolos al mismo
    tiempo: webmail SMTP(quien levanta los mails) con HTTP (para ver la página
    web).  Los paquetes acá se denominan mensajes, que son los que yo quiero
    transmitir. 

    Capa transporte: la forma en como se transportan los mensajes. 2 tipos de
    Servicio: Orientado a conexión(TCP) y servicio sin conexión (UDP). Los
    protocolos tienen determinada relación, dependiendo el servicio que la capa
    aplicación quiere hacer, por ejemplo si la aplicación quiere enviar
    streaming utilizaría UDP, pero en cambio si quiere utilizar mensajes puede
    utilizar TCP. 

    Capa Red: tiene como objetivo el ruteo de fuentes a destino. Protocolo más
    conocido es el IP que es de enrutamiento. Los paquetes se llaman datagramas.
    ¿Qué hace? asegurarse que los paquetes sigan una determinada ruta para que
    lleguen a destino. Se hace una ponderación por el camino más corto que tiene
    en cuenta la cantidad de nodos por los que va a pasar : cantidad de saltos
    que se pueda tener.  Se debe generar un protocolo que se estime la cantidad
    de saltos que debe hacer. Es lo que se hace en la capa de red, es parte de
    lo que hace el protocolo IP. El datagrama se engia al Enalce

    Capa Enlace: Se encarga de la transferencia de los paquetes de la capa de
    red entre nodos vecinos. En la de enlace lo que hace es establecer el enlace
    entre nodos del camino establecido por la capa de Red. 
    La independicia de capas sirve par que en caso d eque se cambie un enlace
    por ejemplo en el camino, no importa para la capa de red, solo le es
    pertinente para la de enlace. 
    Los paquetes se llaman tramas. 
    Protocolos: PPP, Ethernet, Wifi

    Dispositivos: 
    Cada dispositivo puede procesar cierta capa, por ejemplo el modem que es de
    capa física no puede correr un protocolo de enlace. 
    Los dispositivos de la capa de abajo NO pueden interpretar los de arriba,
    pero los de arriba SI pueden interpretar los de abajo suyo. (Como en la
    filmina). Porque la información debe pasar por esas capas y por lo tanto lo
    debe saber interpretar. 

    ¿Cómo se hacne para correr los procotoclos y que cada capa cumpla su
    objetivo? La primer capa agarra el dato que quiere mandar una palabra, el
    dato es "Hola". La capa de aplicación le agrega un encabezado donde se
    determina algo para ser interpretada en la capa de aplicación, destino, tipo
    de mensaje, etc. Le agrega su encabezado y se la pasa a la capa de
    transporte y agarra el Hola y el encabezado de aplicación como si fueran
    datos, para esta capa todo lo de la aplicación es dato, lo que hace es
    agregarle su propio encabezado. Con ello se asegura la independencia de
    datos entre datos. Cada capa agrega su encabezado y toma como dato todos los
    encabezados de la capa inferior. 
    La capa física sólo entiende de 0 y 1. No le importa absolutamente nada, no
    distingue paquetes, lo único que hace es convertir todo eso de datos en
    señales. 
    El switch sólo lee datos de capa de enlace. 
    En cambio el router si puede trabajar sobre los encabezados de capa 3. 

    A la inversa llega el mensaje completo y cada capa le va sacando su propio
    encabezado para su propia capa, hasta que llegue hasta la capa de
    aplicación.
